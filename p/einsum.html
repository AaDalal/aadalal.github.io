Einsum

The basic idea for einstein notation is to drop the ∑\sum∑ from summations in some cases (reducing notational complexity). For instance, you might want to compute the matrix multiplication between a row vector and a column vector: ∑ixi∗yi\sum_i x_i * y_i∑i​xi​∗yi​ is xiyix_i yixi​yi in einsum.

According to Wolfram MathWorld there are 3 rules:

  1. Repeated indices are implicitly summed over.

The index is just iii or jjj or the like. Repeated indices mean they appear more than once in a single term.

Examples:

  • xiyi=∑ixiyix_i y_i = \sum_i x_i y_ixi​yi​=∑i​xi​yi​ ☑️
  1. Each index can appear at most twice in any term.

This means that even the indices you intend to sum over should be repeated at most twice (obviously indices you don't want to sum over should be repeated at most once)

Examples:

  1. xiyizix_i y_i z_ixi​yi​zi​ ✖️
  2. xiyjzix_i y_j z_ixi​yj​zi​ ☑️
  1. Each term must contain identical non-repeated indices.

This is a rule that only applies when you have more than one term. It specifies that if you use a term that only appears once in the first term (a non-repeating term), then you should make an effort to use the same index where there are non-repeating indices. To my mind, this does not always make things intuitive (see the examples below), and it is not emphasized among many Einstein notation users.

Examples:

  1. xiyizj+xicpx_i y_i z_j + x_i c_pxi​yi​zj​+xi​cp​ ✖️
  2. xiyizj+xicjx_i y_i z_j + x_i c_jxi​yi​zj​+xi​cj​ ☑️

Does this imply that the index jjj in zjz_jzj​ equals the index jjj in cjc_jcj​? It's a little unclear to my eye.

Applications of Einstein Notation

Kronecker Delta

The Kronecker delta is a pretty simple idea that says the following (written as pseudocode)

δij\delta_{ij}δij​:

if i == j:
    1
else: 
    0

It allows a neat rewriting of the dot product as δijxiyj\delta_{ij} x_i y_jδij​xi​yj​, or the trace as δijAij\delta_{ij} A_{ij}δij​Aij​. It also allows a nice expression for matrix multiplication δjkAijBki\delta_{jk} A_{ij} B_{ki}δjk​Aij​Bki​.

Levi-Civita Permutation Tensor

The ϵ\epsilonϵ tensor (called so because it can be described as N-D array) helps compute cyclic things, which arise in cross-products and determinants.

in 3d we can describe the tensor as follows:

ϵijk\epsilon_{ijk}ϵijk​

if i == j or j == k or i == k:
    0
else if you can shift ijk s.t. they are in order decreasing order:
    1
else:
    -1

By shift we mean going from a sequence abcabcabc -> bcabcabca -> cabcabcab. Case 2 is called the cyclic or even case, and case 3 is the acyclic/odd case.

The Levi-Civita tensor allows for a few nice things:

  1. It encodes that i≠ji \neq ji=j and i≠ki \neq ki=k since terms go to 0 when this is violated.
  2. The signs of the 2 products in each component CiC_iCi​ are flipped since we have ijkijkijk and ikjikjikj, one of which must necessarily be odd and one even.
